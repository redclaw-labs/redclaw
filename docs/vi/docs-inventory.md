# Bảng kiểm kê tài liệu RedClaw

Bảng kiểm kê này phân loại tài liệu theo mục đích để người đọc có thể nhanh chóng phân biệt giữa tài liệu “hợp đồng runtime hiện hành” và các đề xuất thiết kế.

Rà soát lần cuối: **18 tháng 2, 2026**.

## Chú giải phân loại

- **Hướng dẫn/Tra cứu hiện hành**: dự kiến khớp với hành vi runtime hiện tại
- **Chính sách/Quy trình**: quy tắc cộng tác hoặc quản trị
- **Đề xuất/Lộ trình**: khám phá thiết kế; có thể chứa lệnh giả định
- **Ảnh chụp nhanh (Snapshot)**: báo cáo vận hành theo mốc thời gian

## Khuyến nghị bảo trì

1. Cập nhật `commands-reference` mỗi khi bề mặt CLI thay đổi.
2. Cập nhật `providers-reference` khi catalog/alias/env vars của provider thay đổi.
3. Cập nhật `channels-reference` khi hỗ trợ kênh hoặc ngữ nghĩa allowlist thay đổi.
4. Giữ các snapshot có dấu thời gian và không chỉnh sửa lại nội dung.
5. Gắn nhãn tài liệu đề xuất rõ ràng để tránh bị hiểu nhầm như hợp đồng runtime.
6. Giữ liên kết README/docs-hub giữa các ngôn ngữ được căn chỉnh khi thêm tài liệu cốt lõi mới.
7. Cập nhật `docs/SUMMARY.md` và các collection index mỗi khi thêm tài liệu quan trọng mới.
